# create-web-loom

`create-web-loom` scaffolds a Vite project, installs Web Loom packages, and applies framework-specific starter overlays so your app boots into a Web Loom MVVM counter immediately.

```bash
npm create web-loom@latest
```

## Flow

1. **Vite scaffold**: Runs `create-vite` interactively.
2. **Scaffold detection**: Detects framework + JS/TS variant.
3. **Package install**: Installs published `@web-loom/*` packages.
4. **Starter overlay**: Copies templates from `src/templates/shared` and `src/templates/<framework>/<variant>` into the generated project.

The overlay intentionally replaces Vite's runnable starter entry files.

If a generated scaffold cannot be mapped to a supported template target, the CLI fails fast with an unsupported-scaffold error instead of writing incorrect files.

## Supported frameworks

| Framework | Variants | Runnable files replaced |
|---|---|---|
| React | TS, JS | `src/App.tsx` / `src/App.jsx` |
| Preact | TS, JS | `src/app.tsx` / `src/app.jsx` |
| Vue | TS, JS | `src/App.vue` |
| Solid | TS, JS | `src/App.tsx` / `src/App.jsx` |
| Svelte | TS, JS | `src/App.svelte` |
| Lit | TS, JS | `src/my-element.ts` / `src/my-element.js` |
| Vanilla | TS, JS | `src/main.ts` / `src/main.js` |
| Qwik | TS-first | `src/routes/index.tsx` |

Shared starter files are copied for every framework (for example `src/viewmodels/CounterViewModel.ts`).

## Template source of truth

Templates are file-based:

- `packages/create-web-loom/src/templates/shared/**`
- `packages/create-web-loom/src/templates/<framework>/<variant>/**`

During package build, these are copied to `dist/templates/**`. Runtime injection uses `dist/templates` so published binaries remain self-contained.

## Framework and variant detection

Framework detection checks generated dependencies in this priority order:

1. `@builder.io/qwik`
2. `solid-js`
3. `preact`
4. `react`
5. `vue`
6. `lit`
7. `svelte`
8. fallback: `vanilla`

Variant detection resolves `ts | js` from:

- `typescript` dependency/devDependency, and
- scaffold entry file extensions (`.ts/.tsx` vs `.js/.jsx`).

## Usage

```bash
npm create web-loom@latest
npm create web-loom@latest my-app
```

After generation:

```bash
cd my-app
npm install
npm run dev
```
